1. 컨텍스트 스위칭에 대해 설명해보세요.
> 컨텍스트 스위칭은 CPU가 현재 실행 중인 작업의 상태를 저장하고,
다른 작업의 상태를 불러와 실행 대상을 바꾸는 과정입니다.

이 개념은
여러 프로세스나 스레드를 하나의 CPU 또는 제한된 CPU 자원 위에서
번갈아 실행해야 하는 문제를 해결하기 위해 필요합니다.

핵심 특징은
첫째, 현재 작업의 레지스터 값, 프로그램 카운터, 스택 정보 같은 실행 문맥을 저장하고 복원해야 한다는 점이고,
둘째, 이 과정에서 실제 업무를 처리하는 것이 아니라 전환 작업 자체에 비용이 든다는 점이고,
셋째, 멀티태스킹 환경에서 공정한 CPU 분배와 동시성 처리의 기반이 된다는 점입니다.

그래서 실제로는
운영체제가 여러 프로세스를 번갈아 실행할 때,
또는 멀티스레드 환경에서 실행 흐름을 전환할 때 사용됩니다.
