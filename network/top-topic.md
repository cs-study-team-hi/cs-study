1. TCP와 UDP의 차이점에 대해서 설명해보세요.
초기 답변)
우선 둘다 5계층 기준으로 4번째 레이어인 트랜스포트 레이어로 애플리케이션을 지나가기 전 단계인 프로토콜입니다.

이 때 tcp와 같은 경우, 혼잡 제어, 에러 제어, 흐름 제어 등을 보장하기 위해 여러 규칙을 넣었지만
udp와 같은 경우는 저런 problem에 대한 완전한 보장을 프로토콜 자체만으로 하지 않음

tcp는 세그먼트가 보장을 하려다 보니 두꺼움
udp는 상대적으로 얇아졌다.

데이터 손실의 영향이 강력한 상황에서는 tcp를 사용하고, 동영상 시청과 같이 데이터의 손실이 약간 허용되면 udp를 사용하면 됩니다.

> 깔끔한 답변
> TCP와 UDP는 모두 전송 계층 프로토콜이고, 핵심 차이는 신뢰성과 속도의 트레이드오프입니다.
TCP는 연결 지향이며 순서 보장, 재전송, 흐름 제어, 혼잡 제어를 통해 신뢰성 있는 전송을 제공합니다.
반면 UDP는 비연결형이라 별도의 보장 기능이 적어서 더 가볍고 빠릅니다.
그래서 정확한 전달이 중요하면 TCP, 지연 최소화가 중요하면 UDP를 사용합니다.
> 

2. GET과 POST의 차이점에 대해서 설명해보세요.
> 우수 답변
> GET과 POST는 둘 다 HTTP 요청 방식이라는 공통점이 있습니다.

다만 차이는 크게 세 가지가 있습니다.

첫째, 목적 측면에서  
GET은 서버에서 데이터를 조회할 때 주로 사용되고,  
POST는 서버에 새로운 데이터를 전달하거나 생성할 때 주로 사용됩니다.

둘째, 데이터 전달 방식 측면에서  
GET은 보통 데이터를 URL의 쿼리 파라미터에 담아 보내고,  
POST는 보통 데이터를 HTTP Body에 담아 보냅니다.

셋째, 특성 측면에서  
GET은 같은 요청을 여러 번 보내도 서버 상태를 바꾸지 않는 조회에 적합하고,  
POST는 요청에 따라 서버 상태가 바뀔 수 있어서 데이터 등록이나 처리에 적합합니다.

그래서 단순 조회가 중요하면 GET을,  
데이터 생성이나 변경 처리가 중요하면 POST를 선택합니다.

3. HTTP와 HTTPS의 차이점에 대해서 설명해보세요.

둘 다 웹에서 클라이언트와 서버가 통신하기 위한 프로토콜이라는 공통점이 있습니다.

다만 차이는 크게 세 가지가 있습니다. 첫째, 보안 측면에서 HTTP는 데이터를 평문으로 주고받기 때문에 중간에서 내용을 볼 수 있고 변조될 위험도 있지만, HTTPS는 SSL/TLS를 통해 데이터를 암호화해서 전송하므로 도청이나 변조를 방지할 수 있습니다.
둘째, 동작 방식 측면에서 HTTP는 요청과 응답을 바로 주고받는 반면, HTTPS는 통신 전에 TLS 핸드셰이크를 통해 서버 인증서를 검증하고 암호화에 사용할 세션을 만든 뒤 데이터를 주고받습니다.
셋째, 성능과 사용 상황 측면에서 HTTP는 구조가 단순해서 약간 가볍지만, HTTPS는 현대 환경에서는 성능 부담이 많이 줄었고 보안이 중요한 운영 서비스에서는 사실상 기본입니다.
그래서 단순한 내부 테스트나 민감하지 않은 통신이면 HTTP를 사용할 수 있지만, 로그인, 결제, 개인정보 처리처럼 보안과 신뢰가 중요한 상황에서는 HTTPS를 선택하는 것이 맞습니다.
